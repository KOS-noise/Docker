## 모듈 01. 프로세스 관리

### 1-1. 프로세스 확인 ⭐

```bash
# 현재 실행 중인 전체 프로세스
ps aux

# 특정 프로세스 찾기 ⭐
ps aux | grep java
ps aux | grep python
ps aux | grep nginx
```

**ps aux 출력 읽는 법**

```
USER       PID  %CPU %MEM    VSZ   RSS  STAT  COMMAND
ubuntu    1234   2.1  5.3  812044 108932  Sl   java -jar myapp.jar
```

| 컬럼 | 의미 |
|------|------|
| `USER` | 프로세스 실행 계정 |
| `PID` | 프로세스 ID (종료 시 사용) |
| `%CPU` | CPU 사용률 |
| `%MEM` | 메모리 사용률 |
| `STAT` | 상태 (S=슬립, R=실행, Z=좀비) |
| `COMMAND` | 실행 명령어 |

### 1-3. 프로세스 종료 ⭐

```bash
# 정상 종료 (프로세스가 정리 작업 후 종료)
kill 1234

# 강제 종료 (응답 없을 때 사용)
kill -9 1234

# 프로세스 이름으로 종료 ⭐
pkill python3
pkill -9 java

# 프로세스 이름으로 PID 확인
pgrep java
pgrep -l java      # PID + 이름 함께 출력
```

> **kill vs kill -9**
> - `kill` (SIGTERM): "정리하고 종료해" → DB 커넥션 반환, 파일 닫기 등 처리
> - `kill -9` (SIGKILL): "즉시 강제 종료" → 정리 없이 죽임, 데이터 손실 가능
> - 항상 `kill` 먼저 시도 → 안 되면 `kill -9`

---

### 1-4. 백그라운드 실행 & nohup ⭐

```bash
# & : 백그라운드로 실행 (터미널 유지 중에만 동작)
python3 -m http.server 8080 &

# nohup : 터미널이 끊겨도 계속 실행 ⭐
nohup python3 -m http.server 8080 &

# nohup 로그 확인
tail -f nohup.out

# 백그라운드 작업 목록
jobs
```

> **& vs nohup**
> - `&` 만 쓰면 터미널 종료 시 프로세스도 같이 죽음
> - `nohup` 은 터미널 끊겨도 프로세스 유지 → 실제 배포 시 사용

---

### 1-5. 좀비 프로세스

```bash
# 좀비 프로세스 확인 (STAT이 Z인 것)
ps aux | awk '$8 == "Z" {print}'
```

> 좀비 프로세스는 이미 종료됐지만 부모 프로세스가 처리를 안 한 상태.
> `kill -9` 로 죽일 수 없다. 부모 프로세스를 재시작하거나 서버를 재부팅해야 한다.

## 모듈 02. 포트 관리

### 2-1. 포트 사용 현황 확인 ⭐

```bash
# 모든 리스닝 포트 확인 ⭐
ss -tulpn

# netstat (오래된 방식이지만 여전히 많이 씀)
netstat -tulpn
```

**ss -tulpn 출력 읽는 법**

```
Netid  State   Local Address:Port   Process
tcp    LISTEN  0.0.0.0:8080         users:(("java",pid=1234,fd=10))
tcp    LISTEN  0.0.0.0:3306         users:(("mysqld",pid=567,fd=20))
tcp    LISTEN  0.0.0.0:22           users:(("sshd",pid=890,fd=3))
```

| 옵션 | 의미 |
|------|------|
| `-t` | TCP |
| `-u` | UDP |
| `-l` | LISTEN 상태만 |
| `-p` | 프로세스 정보 표시 |
| `-n` | 이름 대신 숫자로 표시 |

---

### 2-2. 특정 포트 사용 프로세스 찾기 ⭐

```bash
# 8080 포트를 사용 중인 프로세스 확인 ⭐
lsof -i :8080

# ss로 확인
ss -tulpn | grep :8080

# 포트 번호로 PID만 추출 ⭐
lsof -t -i :8080
```

**lsof -i :8080 출력 예시**

```
COMMAND  PID   USER   TYPE  DEVICE SIZE/OFF NODE NAME
java    1234 ubuntu  IPv4   12345      0t0  TCP *:8080 (LISTEN)
```

---

### 2-3. 포트 충돌 해결 ⭐

```
에러 메시지: "Address already in use: bind"
            "Port 8080 is already in use"
```

```bash
# 1. 어떤 프로세스가 점유 중인지 확인
lsof -i :8080

# 2. PID 확인 후 종료
kill 1234

# 3. 한 번에 해결 ⭐
kill $(lsof -t -i :8080)

# 4. 포트 해제 확인
ss -tulpn | grep :8080
```

---

### 2-4. 백엔드 개발자 자주 쓰는 포트 번호

| 포트 | 용도 |
|------|------|
| `8080` | Spring Boot / WAS 기본 |
| `3000` | Node.js / Next.js |
| `3306` | MySQL |
| `5432` | PostgreSQL |
| `6379` | Redis |
| `27017` | MongoDB |
| `80` | HTTP |
| `443` | HTTPS |
| `22` | SSH |

## 실습 시나리오 (25분)

> **상황**: v1 앱이 8080에서 운영 중인데 v2를 배포하려다 포트 충돌 발생

```
v1 실행 중 (8080)
      ↓
v2 배포 시도 → 에러
      ↓
원인 파악 → v1 종료
      ↓
v2 배포 성공
```

---

### Step 1. v1 앱 실행 (운영 중인 상황 재현)

```bash
# v1 앱을 nohup으로 백그라운드 실행
nohup python3 -m http.server 8080 &

# 실행 확인
ps aux | grep python
ss -tulpn | grep :8080
```

---

### Step 2. v2 배포 시도 → 포트 충돌 에러 확인

```bash
# 같은 8080 포트로 v2 실행 시도
python3 -m http.server 8080

# 에러 발생:
# OSError: [Errno 98] Address already in use
```

---

### Step 3. 원인 파악

```bash
# 어떤 프로세스가 8080을 점유 중인지 확인
lsof -i :8080

# 프로세스 상세 정보
ps aux | grep python
```

---

### Step 4. v1 종료

```bash
# 방법 1: PID 직접 지정
kill 1234          # lsof에서 확인한 PID

# 방법 2: 한 번에 종료 ⭐
kill $(lsof -t -i :8080)

# 종료 확인
ss -tulpn | grep :8080    # 아무것도 안 나오면 성공
ps aux | grep python
```

---

### Step 5. v2 배포 성공

```bash
# v2 앱 실행
nohup python3 -m http.server 8080 &

# 정상 실행 확인
ss -tulpn | grep :8080
curl http://localhost:8080
```

---

### Step 6. top으로 리소스 확인

```bash
top
# M 키: 메모리 사용량 순 정렬
# P 키: CPU 사용량 순 정렬
# q 키: 종료
```

## 핵심 정리

| 상황 | 명령어 |
|------|--------|
| 특정 프로세스 찾기 ⭐ | `ps aux \| grep 이름` |
| 프로세스 종료 ⭐ | `kill PID` |
| 프로세스 강제 종료 ⭐ | `kill -9 PID` |
| 이름으로 종료 ⭐ | `pkill 이름` |
| 터미널 끊겨도 실행 유지 ⭐ | `nohup 명령어 &` |
| 전체 포트 확인 ⭐ | `ss -tulpn` |
| 특정 포트 확인 ⭐ | `lsof -i :포트번호` |
| 포트 점유 프로세스 바로 종료 ⭐ | `kill $(lsof -t -i :8080)` |
| 실시간 모니터링 | `top` |
