# 4교시. 서비스간 연결 확인
> 백엔드 개발자를 위한 리눅스 실무 | 50분 | 이론 20분 + 실습 30분

---

## 실습 환경

이 실습은 두 개의 컨테이너가 서로 통신하는 구조로 진행됩니다.

```bash
# 1. API 서버 실행 (httpbin: 요청을 거울처럼 보여주는 테스트 서버)
docker run -d --name api-server kennethreitz/httpbin

# api-server IP 확인 (curl 호출 시 사용)
docker inspect api-server | grep '"IPAddress"' # 리눅스 용
docker inspect api-server | Select-String "IPAddress" #window 용 1
docker inspect api-server | findstr "IPAddress" #window 용 2

# 2. 실습용 우분투 컨테이너 실행 (privileged 권한 필요 - ufw 실습용)
docker run -it --privileged --name linux-lab_ver02 ubuntu:22.04 /bin/bash
sed -i 's/archive.ubuntu.com/ftp.kaist.ac.kr/g' /etc/apt/sources.list

# [컨테이너 안에서] 필수 패키지 설치
apt update && apt install -y curl wget iputils-ping traceroute net-tools ufw
```

> `docker inspect` 로 확인한 IP를 모든 명령의 `172.17.0.2` 대신 실제 IP로 사용한다.
> 예: `172.17.0.2` 이면 `curl http://172.17.0.2/get`

---

## 학습 목표

1. curl로 API 요청을 보내고 응답을 분석할 수 있다.
2. ping/traceroute로 서버간 네트워크 연결을 진단할 수 있다.
3. ufw로 방화벽을 설정하고 특정 목적지(API 서버)로의 통신을 제어할 수 있다.

## 모듈 04. curl — API 연결 테스트 (10분)

### 4-1. 기본 요청 (GET)

`httpbin` 서버의 `/get` 경로는 내가 보낸 요청 정보를 그대로 보여줍니다.

```bash
# 기본 GET 요청
curl http://172.17.0.2/get

# 응답 헤더 포함 출력 (-i)
curl -i http://172.17.0.2/get

# 헤더만 출력 (-I)
curl -I http://172.17.0.2/get
```

### 4-2. 데이터 전송 (POST)

```bash
# JSON 바디로 POST (-d)
curl -X POST http://172.17.0.3/post -H "Content-Type: application/json" -d '{"name": "홍길동", "email": "hong@example.com"}'
```

---

### 4-3. 실무 필수 옵션

```bash
# Bearer 토큰 인증 헤더 추가
curl -H "Authorization: Bearer my-secret-token" http://172.17.0.2/bearer

# 응답 코드만 확인 (HTTP 200, 404 등)
curl -o /dev/null -s -w "%{http_code}\n" http://172.17.0.2/status/200
curl -o /dev/null -s -w "%{http_code}\n" http://172.17.0.2/status/404

# 타임아웃 설정 (서버 응답이 늦을 때 중단)
curl --connect-timeout 2 http://172.17.0.2/delay/5
```

---

### 4-4. 자주 마주치는 에러 분석

| 에러 메시지 | 주요 원인 | 해결 방법 |
|------|------|------|
| `Connection refused` | 서버 프로세스가 안 떠있거나 포트가 틀림 | `ps` 또는 `ss`로 서버 상태 확인 |
| `Connection timed out` | **방화벽(ufw) 차단** 또는 네트워크 단절 | 방화벽 허용 규칙 확인 |
| `Could not resolve host` | 도메인 이름 해석 실패 | `/etc/hosts` 또는 DNS 확인 (IP 사용 시 발생 안 함) |
| `SSL certificate problem` | 자체 서명 인증서 등 보안 오류 | `-k` (insecure) 옵션 사용 |

## 모듈 05. ping & traceroute — 네트워크 진단 (5분)

### 5-1. ping — 연결 여부 확인

```bash
# API 서버로 3번만 핑 보내기
ping -c 3 172.17.0.2
```

### 5-2. traceroute — 경로 추적

```bash
# 목적지까지 거치는 관문(Hop) 확인
traceroute 172.17.0.2
```

## 모듈 06. ufw 방화벽 — 통신 제어 (5분)

> [!WARNING]
> Docker 컨테이너 내에서 `ufw`를 실습하려면 반드시 `--privileged` 옵션으로 실행해야 합니다.

### 6-1. 기본 정책 설정

```bash
# ufw 활성화 (기본적으로 나가는 것은 허용, 들어오는 것은 차단)
ufw enable

# 기본 정책 확인
ufw status verbose
```

### 6-2. 특정 통신 차단 (Outbound)

실무에서는 보안을 위해 서버가 외부 API로 데이터를 보내는 것(Outbound)도 제한합니다.

```bash
# API 서버(172.17.0.2)로 나가는 통신 차단
ufw deny out to any port 80

# curl 테스트 (Connection timed out 발생함)
curl --connect-timeout 3 http://172.17.0.2/get

# 규칙 삭제 (다시 허용)
ufw delete deny out to any port 80
```

## 실습 과제 (30분)

# 1. API 서버(172.17.0.2)가 호출되는지 확인하세요.
ping -c 2 172.17.0.2
curl http://172.17.0.2/get

### Step 2. API 응답 분석 (httpbin 활용)
다양한 옵션으로 `httpbin` 서버를 호출하고 결과를 분석하세요.
```bash
# 1. POST로 데이터 보내기 (응답의 "json" 항목 확인)
curl -X POST http://172.17.0.2/post -d '{"msg":"hello"}' -H "Content-Type: application/json"

# 2. 의도적으로 500 에러 유도하기
curl -I http://172.17.0.2/status/500

# 3. 3초 지연 후 응답받기 (타임아웃 발생시켜보기)
curl --connect-timeout 2 http://172.17.0.2/delay/3
```

### Step 3. 방화벽 차단 및 해제 실습
`ufw`를 사용하여 특정 서비스로의 접근을 제어해보세요.
```bash
# 1. 모든 Outbound 통신 차단
ufw default deny outgoing

# 2. API 서버 호출 시도 (실패해야 함)
curl --connect-timeout 3 http://172.17.0.2/get

# 3. HTTP(80) 포트만 외부로 나가도록 허용
ufw allow out 80/tcp

# 4. 다시 호출 시도 (성공해야 함)
curl http://172.17.0.2/get
```

| 나가는 포트 차단 | `ufw deny out to any port 포트` |
| 나가는 포트 허용 | `ufw allow out 포트/tcp` |

### Step 1. 테스트용 HTTP 서버 실행

```bash
python3 -m http.server 8080 &
```

### Step 2. curl로 다양한 요청 테스트

```bash
# 기본 GET
curl http://localhost:8080

# 응답 코드 확인
curl -o /dev/null -s -w "응답코드: %{http_code}\n" http://localhost:8080

# 상세 헤더 확인
curl -I http://localhost:8080

# 존재하지 않는 경로 (404 확인)
curl -o /dev/null -s -w "응답코드: %{http_code}\n" http://localhost:8080/없는경로
```

### Step 3. 네트워크 연결 진단

```bash
# 구글 DNS로 네트워크 연결 확인
ping -c 3 8.8.8.8

# 도메인 해석 + 연결 확인
ping -c 3 google.com

# 경로 추적
traceroute -m 5 google.com
```

### Step 4. ufw 방화벽 설정

```bash
# 현재 상태 확인
ufw status

# 8080 포트 허용
ufw allow 8080/tcp
ufw status

# 확인: 허용 후 curl 응답
curl http://localhost:8080

# 8080 포트 차단
ufw deny 8080/tcp
ufw status

# 차단 후 curl 응답 확인 (localhost는 방화벽 우회될 수 있음)
curl http://localhost:8080

# 원복
ufw delete deny 8080/tcp
```
